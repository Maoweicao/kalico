    负载传感器¶

    本文档描述了负载传感器的支持。负载传感器使用模数转换器（ADC）测量 应用于应变计的施加力。它们可用于读取力数据、称量灯丝卷等物体， 或充当探针。

    警告：在使用负载传感器之前，必须进行校准。如果校准不正确， 报告的力/重量将不正确，这可能导致对负载传感器和/或打印机的永久损坏。 此模块无法补偿校准不足、损坏的应变计或电气噪声。

    基本负载传感器配置¶

    可以配置负载传感器用作秤。

    [load_cell]
sensor_type: hx717
sclk_pin: PA5
dout_pin: PA4
sample_rate: 320
counts_per_gram: 245
reference_tare_counts: 12345

    • sensor_type: hx717
      必需
      每个传感器都有不同的必需字段，检查其配置参考了解详情：

      • hx711
      • hx717
      • ads1220
      • ads131m02
      • ads131m04
    • counts_per_gram: 245
      默认值：无
      从原始传感器计数到克的转换系数，由LOAD_CELL_CALIBRATE计算。
    • reference_tare_counts: 12345
      必需
      原始传感器计数中的基线去皮值，由LOAD_CELL_CALIBRATE设置。

    诊断¶

    检查负载传感器操作¶

    LOAD_CELL_DIAGNOSTIC (文档)

    从负载传感器收集样本并报告健康状况和统计信息。 首次连接负载传感器时运行此命令以验证接线和配置。

    LOAD_CELL_DIAGNOSTIC
// 收集负载传感器数据10秒...
// 收集的样本：3211
// 每秒测量的样本：332.0
// 良好样本：3211，饱和样本：0，唯一值：900
// 样本范围：[4.01%至4.02%]
// 样本范围/传感器容量：0.00524%

    检查输出：

    • 每秒测量的样本应接近配置的sample_rate。如果不是，检查配置。 对于HX711，采样率由硬件设置。
    • 饱和样本应为0。非零表示超出传感器测量范围的过度力。
    • 唯一值应是收集样本的很大百分比。如果唯一值为1，验证接线。
    • 在测试期间点击或按下传感器。如果传感器正常工作，样本范围应增加。

    校准¶

    校准负载传感器¶

    LOAD_CELL_CALIBRATE (文档)

    启动交互式校准实用程序。校准过程由三个步骤组成：

    1. TARE - 建立零力值并设置reference_tare_counts
    2. CALIBRATE GRAMS=<value> - 应用已知力并计算counts_per_gram
    3. ACCEPT - 保存校准结果到配置

    随时使用ABORT取消校准。

    运行校准后的LOAD_CELL_DIAGNOSTIC将以克为单位显示其他信息。

    应用已知力¶

    CALIBRATE GRAMS=<value>步骤需要应用已知力。该方法取决于负载传感器位置：

    平台安装的负载传感器（床下或灯丝保持器下）： 在平台上放置已知质量的对象。理想情况下，使用负载传感器额定容量的很大百分比 （例如，5kg传感器使用5kg）。

    工具头负载传感器： 在床上放置数字秤，轻轻降低工具头到其上（或抬起床如果床移动）。 使用至少1kg的力。太大的力可能损坏床或工具头，所以小步骤移动。 从数字秤获取读数并将其输入到CALIBRATE GRAMS=<value>命令。

    了解校准结果¶

    CALIBRATE GRAMS=555
// 校准值：-2.78%（-559467），计数/克：87.944082，总容量：+/- 29.14Kg

    Calibration value:显示使用了传感器范围的多少百分比来校准。

    Counts/gram:是等于1克力的传感器计数数。 此数字越大，秤就越精确。

    Total capacity是传感器可能记录的最高力。 Total capacity应接近负载传感器的额定容量。如果大得多， 考虑更高的增益设置或更灵敏的负载传感器。对于位宽低于24位的传感器， 这更为关键。

    操作¶

    读取力数据¶

    LOAD_CELL_READ

    读取负载传感器上的当前力。

    LOAD_CELL_READ
// 10.6g（1.94%）

    力数据也在load_cell打印机对象中可用：

    {% set grams = printer.load_cell.force_g %}

    此值在最后1秒内平均，类似于温度传感器。

    对负载传感器进行去皮¶

    LOAD_CELL_TARE

    将当前读数设置为零力。用于测量相对重量变化，如打印期间的灯丝消耗。

    LOAD_CELL_TARE
// 负载传感器去皮值：5.32%（445903）

    去皮值在load_cell打印机对象中可用：

    {% set tare_counts = printer.load_cell.tare_counts %}

    负载传感器探针配置¶

    此示例为校准的负载传感器添加探针功能。[load_cell_probe]部分 包括所有[load_cell]参数、负载传感器探针特定参数和 [probe]参数。

    [load_cell_probe]
# 负载传感器设置
sensor_type: hx717 # 传感器特定配置
counts_per_gram: 245
reference_tare_counts: 12345
# 负载传感器探针设置
trigger_force: 75
force_safety_limit: 2000
drift_safety_limit: 1000
drift_filter_cutoff_frequency: 0.5
# 探针设置
z_offset: 0.0
    • sensor_type: hx717
    • counts_per_gram: 245
      与基本负载传感器相同
    • reference_tare_counts: 12345
      默认值：无
      原始传感器计数中的基线去皮值，由LOAD_CELL_CALIBRATE设置。 用作force_safety_limit的零值来定义安全工作范围。
    • trigger_force: 75
      默认值：75（75克）
      在探针期间触发限位开关的力（克），相对于探针开始时的去皮值测量。 预期会产生过冲；更高的探针速度或更低的采样率会增加峰值力。 详见多MCU回零了解多MCU时序考虑。 75克默认值是保守的。较高的值，高达200克，在处理脓性灯丝时可改进性能。
    • force_safety_limit: 2000
      默认值：2000（+/-2Kg）
      要安全启动探针移动，探针上的力必须低于此限制。 这将reference_tare_counts视为其零值。 通过将值设置为0可禁用此检查。如果超过，探针停止并出现错误 !! Load Cell Probe Error: force of 3000g exceeds force_safety_limit (2000g) before probing!。 此值需要足够大以容纳：
      • Bowden管和拖链力在整个打印体积和探针移动长度中变化
      • 整个探针温度范围内的温度漂移
    • drift_safety_limit: 1000
      默认值：1000（+/-1Kg）
      这是探针在探针移动期间允许的最大力。 设置为0禁用此安全检查。如果超过，探针停止并出现错误 !! Load Cell Probe Error: force exceeded drift_safety_limit before triggering!。 使用drift_filter_cutoff_frequency时此安全措施很重要， 因为可能通过太高的截止频率来击败探针触发。
    • drift_filter_cutoff_frequency: 0.5
      默认值：无（禁用）
      连续去皮漂移滤波器的截止频率（Hz）。 启用MCU上的滤波器以跟踪来自Bowden管和拖链的漂移。 需要SciPy。设置此值过高会延迟探针触发 并增加工具头上的力。
    • z_offset: 0.0
      必需
      床和喷嘴之间的距离（毫米），当探针触发时。 对于负载传感器探针，这是0。

    详见配置参考了解所有可用选项。

    安全¶

    负载传感器是直接喷嘴接触探针。系统包括防止工具头上过度用力的安全检查。 配置值选择不当可能会破坏这些保护。

    校准检查： 在回零或探针之前，负载传感器探针检查是否已校准。 如果未校准，打印机停止并出现错误!! Load Cell Probe Error: Load Cell not calibrated。

    准确的counts_per_gram： 此设置将原始计数转换为克。所有安全限制都以克为单位。 不准确的值允许工具头上过度用力。不要猜测此值——始终使用LOAD_CELL_CALIBRATE。

    保守的trigger_force： 探针总是在停止前超过trigger_force。100克的设置可能导致350克峰值力。 过冲随着更快的探针速度、低采样率或多MCU配置增加。

    force_safety_limit保护： 此设置在启动回零或探针前检测探针上的过度用力。 如果超过限制，探针停止并出现错误，例如 !! Load Cell Probe Error: force of 3000g exceeds force_safety_limit (2000g) before probing!。 这可能由以下原因引起：

    • 探针在探针开始之前与床碰撞。例如通过水平移动到倾斜床
    • Bowden管或拖链的过度力
    • 挤出机推动灯丝引起的应变计上的过度力
    • 损坏的应变计，导致零点远离reference_tare_counts值移动

    drift_safety_limit保护： 这设置探针在触发前允许的最大力。如果测得的力变化超过限制， 探针停止并出现错误：!! Load Cell Probe Error: force exceeded drift_safety_limit before triggering!。 这可能由几个原因触发：

    • 在探针时探针被主动加热，导致去皮值漂移
    • 设置drift_filter_cutoff_frequency过高，导致点击事件被滤出
    • 探针期间Bowden管/拖链力的大变化

    看门狗任务： 在回零时，看门狗监视传感器数据。如果传感器无法在2个采样周期内发送测量值， MCU以错误关闭!! Load Cell Probe Error: timed out waiting for sensor data。 这通常表示ADC故障或接地不足。确保框架、电源和打印床接地。 可能需要多个接地连接。对接地接触点进行砂磨氧化铝以获得良好的电气接触。

    点击验证和重试： 探针验证每个点击的形状、断裂接触时序和运动时间顺序。无效的点击被拒绝 （is_valid=False）并根据探针配置的bad_probe_strategy和bad_probe_retries 重试。这防止接受被污染或质量差的点击。详见点击验证错误代码 了解验证失败类型。请注意，验证捕捉许多但不是所有坏点击， 适当的喷嘴温度和清洁度仍然至关重要。

    测试探针操作¶

    LOAD_CELL_TEST_TAP [COUNT=<taps>] [TIMEOUT=<seconds>]
    默认COUNT：3
    默认TIMEOUT：30

    在不移动轴的情况下测试探针操作。在结束前检测指定数量的点击。 如果在超时期间内未检测到点击，命令失败。该命令验证点击质量 并将验证错误记录到控制台。

    注意：负载传感器探针不支持QUERY_ENDSTOPS或QUERY_PROBE， 它们总是返回未触发。使用LOAD_CELL_TEST_TAP在探针前验证功能。

    回零配置¶

    负载传感器探针支持Z轴回零。回零的精度低于使用PROBE命令探针。 回零后，使用PROBE进行高精度Z回零：

    PROBE HOME=Z

    探针温度¶

    在回零和探针期间保持喷嘴温度低于灯丝出油点。 140°C是所有灯丝类型的良好起点。

    灯丝出油是探针错误的主要来源。Kalico验证点击质量 并拒绝许多坏点击（例如由于出油）。防止出油并保持清洁喷嘴仍然是最佳实践。 不建议在打印温度探针。监视控制台 是否存在点击验证错误（例如TAP_SHAPE_INVALID、TAP_BREAK_CONTACT_TOO_LATE）， 这表示点击质量差。

    喷嘴保护¶

    详见Voron Tap的activate_gcode 了解保护打印表面免受热喷嘴的信息。

    喷嘴清洁¶

    在探针前清洁喷嘴。建议的序列：

    1. 加热喷嘴到探针温度（例如，M109 S140）
    2. 回零机器（G28）
    3. 刮擦喷嘴
    4. 热浸床
    5. 执行探针任务（QGL、床网格等）

    喷嘴温度补偿¶

    由于出油，不可能在打印温度探针。探针后加热喷嘴，导致其膨胀。 喷嘴沿其长度膨胀最多，朝向床。这应该用z_thermal_adjust补偿。

    在两个温度（例如180°C和290°C）测量PROBE_ACCURACY并计算：

    temp_coeff = (z_average_hot - z_average_cold) / (temp_hot - temp_cold)

    示例：temp_coeff = -0.05 / (290 - 180) = -0.00045455

    预期负值（z_thermal_adjust将以负值推动喷嘴远离床，以正值推动它）。

    示例配置：

    [z_thermal_adjust nozzle]
temp_coeff: -0.00045455
sensor_type: temperature_combined
sensor_list: extruder
combination_method: max
min_temp: 0
max_temp: 400
max_z_adjustment: 0.1

    床网格设置¶

    禁用relative_reference_index 因为负载传感器探针给出z的绝对值，不相对于任何东西， 不需要relative_reference_index。从[bed_mesh]中简单删除设置。 从配置中删除该行会将其关闭。

    启用激进的移动分割

    move_check_distance: 3.0
split_delta_z: 0.01

    设置网格以尽可能频繁地调整z高度。这两个设置改变床网格如何评估z变化。 最小化split_delta_z以获得高分辨率网格跟踪（0.01是10微米，10倍探针分辨率）。 为move_check_distance选择小值强制床网格更频繁地重新评估z高度。 如果这些设置保留为默认值，您可能在第一层中看到由于不频繁调整引起的条纹。

    启用horizontal_z_clearance

    horizontal_z_clearance: 0.4

    使用horizontal_z_clearance，探针在网格点之间总是缩回该数量。 这可以大大减少z旅行距离，同时适应床形状。更少的旅行距离加快了探针。

    在网格化前使用NOZZLE_CLEANUP

    NOZZLE_CLEANUP

    这点击喷嘴直到它报告3个连续成功的探针，证明喷嘴清洁。 这在网格化前清除喷嘴中的任何出油以获得最佳网格质量。 这应在打印区域外进行。详见NOZZLE_CLEANUP

    为网格化使用CIRCLE策略

    BED_MESH STRATEGY=CIRCLE

    圆圈策略是[probe]的功能。当负载传感器探针检测到被污染的探针时， 它将移动到圆形图案中的相邻位置。对于床网格，此轻微位置偏移 远不如使用被污染的探针关键。重新探针到被污染的位置不太可能成功 且可能导致网格化操作失败。

    高级配置¶

    连续去皮滤波¶

    负载传感器探针支持MCU上的滤波器，用于补偿来自Bowden管和脐带等 外部力的漂移。如果探针在接触床前触发，这可能是原因。 这有时称为连续去皮，适用于在探针期间经历可变外部力的工具头安装传感器。

    安装SciPy¶

    滤波器默认关闭。需要SciPy库从配置值计算滤波系数。 它需要安装在klipper虚拟环境中。通常：

    ~/klippy-env/bin/pip install scipy

    32位树莓派系统上提供Python 3的预编译版本。

    滤波器调整¶

    drift_filter_cutoff_frequency参数应基于正常操作期间观察到的漂移来选择。

    基本调整指南：

    • 从drift_filter_cutoff_frequency: 0.5 Hz开始
    • Prusa使用0.8 Hz（MK4）和11.2 Hz（XL）；此范围对于实验合理
    • 仅在消除Bowden管漂移后增加
    • 设置过高导致缓慢触发和过度力
    • 保持trigger_force低（默认75克）；漂移滤波器将内部读数保持在零附近
    • 在调整期间保持force_safety_limit保守（默认2kg）
    • 在调整期间保持drift_safety_limit保守（默认1kg）
    • 注意：过度激进的drift_filter_cutoff_frequency可能扭曲点击形状和时序， 触发验证失败（例如，TAP_BREAK_CONTACT_TOO_LATE）。如果出现此类错误， 降低截止频率或探针速度。

    其他滤波参数的调整超出本文档的范围。 在scripts/filter_workbench.ipynb中提供Jupyter笔记本 及其详细分析的示例。

    点击验证¶

    点击探针简介

    负载传感器探针的工作方式与其他探针不同，它在构建表面上"点击"。 探针与构建表面接触后，它向远离构建表面的方向移动一小段，称为回缩移动。 这种向下/向上运动的组合称为"点击"。完整的点击序列被分析 并从原始力数据构建表示。此表示是通过线连接的一系列点。 这是典型点击的图，清晰标记了运动阶段：

    图1：点击阶段¶

    图1 —— 有效点击阶段。图表显示测得的力（黑线）随时间变化的拟合验证线（红色）。 三个阶段被颜色编码：探针（蓝色），其中力随着喷嘴与床接触而上升，停留（绿色）， 其中力在触发后稳定，回缩（红色），其中力在回缩移动抬起喷嘴时返回基线。 垂直线标记阶段边界：蓝色（探针结束/停留开始），绿色（停留结束/回缩开始）， 红色（回缩结束）。理想的点击在探针期间显示陡峭的上升，停留期间的稳定力， 以及回缩期间的清晰返回基线。

    图中的每条线都有名称：

    图2：点击段¶

    图1a —— 显示命名点击线的点击图：方法（触发前的平坦基线），压缩（力积累时的陡峭上升）， 停留（探针稳定时的稳定高力），减压（回缩期间的力下降），以及出发（回缩完成时返回基线）。 彩色垂直线标记阶段之间的边界。

    减压线和出发线的交点报告为探针的Z=0点。这是图上最关键的点。

    回缩移动非常小（~0.2mm）且非常缓慢。由于其缓慢速度，减压线的斜率比压缩线更浅。 这改进了探针的精度，因为力随时间变化较少，意味着探针的z分辨率增加。 实际上，回缩移动是床在一个点的高分辨率力扫描。回缩移动由配置中的pullback_speed 和pullback_dist选项控制。默认设置扫描每微米1个ADC样本，给探针 预期的1微米分辨率。

    基于图的形状，可以判断探针是否良好。探针对点的顺序 和形成的形状进行一些基本检查。如果它不是"点击"形状，则探针报告为不好。 详见点击验证错误代码了解验证失败。

    点击验证错误代码¶

    当检测到质量差的点击时，记录特定错误代码。大多数这些错误可能是喷嘴被污染的症状， 但有些可能表示配置或设置问题：

    错误代码 描述 常见原因
    TAP_CHRONOLOGY 点击图上的4个点在时间上顺序不对 被污染：数据太扭曲，看起来不像点击
    TAP_SHAPE_INVALID 点击形状的其中一个段不往预期方向进行 被污染：数据太扭曲，看起来不像点击
    TAP_BREAK_CONTACT_TOO_EARLY 断裂接触在回缩移动早期检测到 pullback_distance太长。
    TAP_BREAK_CONTACT_TOO_LATE 断裂接触在回缩移动晚期检测到 pullback_distance太短。
    TAP_PULLBACK_TOO_SHORT 喷嘴在回缩移动期间从未与床断裂接触 pullback_distance太短。
    COASTING_MOVE_ACCELERATION 探针移动在探针触发前开始减速 Z未配置负min_position以允许探针移动到z=0以下。例如：position_min: -5。
    TOO_FEW_PROBING_MOVES 梯形移动少于预期 这不常见
    TOO_MANY_PROBING_MOVES 梯形移动多于预期 这不常见

    点击质量¶

    除了基本点击形状检查外，称为点击质量分类器的模块 给每个点击一个0到100的质量分数。分类器的主要目标是 区分可以使用的清洁点击和不能使用的脓性点击。

    分类器使用比率度量使其在不同打印机之间更可转移。 它使用压缩线中总力的比率，因为这是最佳参考度量。 这允许其他数量与压缩力缩放。而绝对度量（角度、力） 对单个物理工具头设计和探针配置效果很好， 但改变这些东西时会中断。

    点击质量组件¶
    组件 描述 原因
    方法力 方法线中力相对于压缩力的变化。预期接近0。 方法线中的大力与在击中床前击中熔融塑料相关联
    出发力 出发线中力相对于压缩力的变化。预期接近0。 喷嘴应在自由空气中，所以任何扭曲通常是由出油引起的。
    基线力 喷嘴与床接触的点和断裂接触的点之间的力差相对于压缩力。预期接近0。 您期望一个秤在取下重物时读零。大差异意味着施加了意外力。
    停留力下降 停留期间力下降相对于压缩力。 一些下降并不罕见，但大下降与塑料从喷嘴和床之间挤出相关。
    标准化减压角度 减压线的斜率与理想减压斜率的匹配程度。标准化为（实际-预期）/预期 出油可以拉动喷嘴，改变斜率。这毁坏了测量的准确性。

    这些因素都结合起来给最终质量分数。只有需要在打印机上测量的组件 是标准化减压角度。

    每个组件都有最大截止值。如果组件超过截止值， 点击质量分数降到0%。每个值是压缩力的百分比：

    组件 阈值 配置参数
    方法力 50% max_approach_force=50
    出发力 25% max_departure_force=25
    基线力 25% max_baseline_force_delta=25
    停留力下降 75% max_dwell_force_drop=75

    点击质量错误代码¶

    如果默认点击质量分类器处于活跃，它可能报告其他错误代码：

    错误代码 描述 常见原因
    LOW_COMPRESSION_FORCE 计算的压缩力小于trigger_force 被污染：喷嘴上的热塑料导致力上升非常缓慢。探针非常软的东西。
    LOW_TAP_QUALITY 点击质量低于min_tap_quality设置 被污染：点击功能可识别但扭曲。配置的min_tap_quality过低。

    所有验证错误都被记录用于故障排除。

    开发者注释¶

    本部分涵盖了用负载传感器探针支持开发工具头板的指导。

    点击分析和验证¶

    负载传感器探针在每次探针尝试上执行完整的点击分析。样本被分析以识别点击点 并构造代表方法、压缩、停留、减压和出发阶段的线。使用穷举肘部查找算法识别点， 并用线性回归构造线。然后计算这些线的交点产生一组点。

    详见图2和图3

    接下来执行基本的健全性检查集：

    1. 运动时间顺序：点被检查以确保在时间上有顺序。由于它们基于力数据、 线性回归和交点，它们可能无序。

    2. 形状验证：点被检查以确保它们形成"点击"形状的图。 这可以是正梯形或负梯形形状。

    3. 断裂接触时序：探针验证断裂接触时间发生在回缩移动的中间三分之二内。 过早或过晚使分析不太可靠。

    如果这些检查中的任何一个失败，探针被标记为坏。如果全部通过，配置的 TapClassifierModule被调用以进一步决定点击是好是坏。内置分类器称为 TapQualityClassifier并通过校准decompression_angle启用。

    自定义点击分类器：可以通过tap_classifier_module配置值 用自定义实现完全替换内置TapQualityClassifier。分类器接收TapAnalysis对象 并可执行附加验证或修改点击位置计算。使用适当的数据集， 可以使用机器学习技术（例如决策树） 构建针对特定打印机硬件定制的更准确的点击分类器。

    失败点击的视觉示例¶

    快速诊断，将您的点击迹线与图1（有效点击）和 下面的失败示例进行比较。将图案与上表中的错误代码匹配。

    图2：低减压力¶

    图2 —— 低减压力（LOW_DECOMPRESSION_FORCE）。构建工作表没有与加热床牢固接触 （塑料碎片工作表），导致停留阶段力大幅下降。减压力与触发力相比太低。

    图3：主要塑料被污染¶

    图3 —— 主要塑料被污染。喷嘴上的软塑料导致峰值力大幅下降 并在停留期间继续力衰减，因为塑料从喷嘴和床之间挤出。 这可能导致LOW_DECOMPRESSION_FORCE错误。

    图4：回缩粘附¶

    图4 —— 轻微塑料粘附。喷嘴孔内的出油塑料在回缩阶段导致轻微力浸（圆形）。 塑料随着它从构建表面抬起而拉动喷嘴。此点击通过验证，因为异常很小， 但表明喷嘴温度可能过高或塑料正在出油。

    图5：基线不一致¶

    图5 —— 基线力不一致（BASELINE_FORCE_INCONSISTENT）。 喷嘴上的塑料导致"缓慢碰撞"在方法线中可见为正斜率。 压缩角远小于90度，基线力在方法和出发之间差异很大。

    ADC传感器选择¶

    推荐的传感器特性：

    • 至少24位分辨率
    • SPI通信
    • 数据就绪（DRDY）引脚用于样本就绪指示，无需SPI查询
    • 可编程增益放大器，128倍增益，消除外部放大器
    • SPI重置指示以检测传感器重启，这是电气问题的常见指示
    • 可选采样率在350 Hz和2 kHz之间（低于250 Hz的速率需要缓慢探针速度并增加工具头力）
    • 对于具有多个负载传感器的床下应用，使用具有所有通道同步采样的ADC， 如ADS131M04。多路ADC具有通道切换后的稳定延迟 和读数时间涂抹问题，降低精度。

    Klipper的bulk_sensor和load_cell_probe基础架构简化了对新传感器的支持。 传感器可以从Python配置，最小采样循环用C编写。

    电源滤波¶

    使用比ADC制造商规格更大的电容器。ADC数据表假设低噪声电池供电环境。 3D打印机产生大量5V总线噪声。使用典型电源和活跃步进驱动器在最终确定电容器值前测试传感器。

    ADC芯片和负载传感器应由LDO驱动。开关降压转换器不适合此应用。

    接地¶

    ADC芯片容易受到噪声和ESD的影响。在芯片下的第一块板层上使用大接地平面。 将芯片远离电源部分和DC-DC转换器。确保到DC电源的适当接地。

    HX711和HX717注释¶

    这些传感器受欢迎但有局限性：

    • 位设置通信有高MCU开销；SPI传感器更有效
    • 无法与MCU通信重置事件，隐藏电气故障
    • HX717（320 Hz）强烈首选于HX711（80 Hz）探针；限制HX711探针速度至2毫米/秒
    • HX711采样率由硬件配置，不可软件配置；10 SPS版本必须重新布线为80 SPS
