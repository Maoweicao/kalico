# 限位开关相位

本文档描述了 Kalico 的步进相位调整限位开关系统。
此功能可以提高传统限位开关的精度。
当使用具有运行时配置的 Trinamic 步进电机驱动器时，它最有用。

典型的限位开关的精度约为 100 微米。（每次轴归位时，
开关可能触发的时间略早或略晚。）尽管这是一个相对较小的错误，
但它可能导致不必要的伪影。特别是，此位置偏差在打印对象的第一层时可能很明显。
相比之下，典型的步进电机可以获得明显更高的精度。

步进相位调整限位开关机制可以使用步进电机的精度来提高限位开关的精度。
步进电机通过循环一系列相位直到完成四个"整步"而移动。
因此，使用 16 微步的步进电机将有 64 个相位，
当向正方向移动时，它将循环通过相位：0、1、2、...61、62、63、0、1、2 等。
至关重要的是，当步进电机在直线导轨上的特定位置时，
它应始终处于相同的步进相位。因此，当滑架触发限位开关时，
控制该滑架的步进器应始终处于相同的步进电机相位。
Kalico 的限位开关相位系统将步进相位与限位开关触发结合起来以提高限位开关的精度。

为了使用此功能，有必要能够识别步进电机的相位。
如果使用 Trinamic TMC2130、TMC2208、TMC2224 或 TMC2660 驱动器
在运行时配置模式下（即不是独立模式），则 Kalico 可以从驱动器查询步进相位。
（如果可以可靠地重置步进驱动器，也可以在传统步进驱动器上使用此系统——请参阅下面的详细信息。）

## 校准限位开关相位

如果使用具有运行时配置的 Trinamic 步进电机驱动器，
则可以使用 ENDSTOP_PHASE_CALIBRATE 命令校准限位开关相位。
首先在配置文件中添加以下内容：
```
[endstop_phase]
```

然后 RESTART 打印机并运行 `G28` 命令，然后运行 `ENDSTOP_PHASE_CALIBRATE` 命令。
然后将工具头移到新位置并再次运行 `G28`。尝试从多个不同位置移动工具头并从每个位置重新运行 `G28`。
至少运行五个 `G28` 命令。

执行完上述操作后，`ENDSTOP_PHASE_CALIBRATE` 命令通常会报告步进器的相同（或几乎相同）相位。
此相位可以保存在配置文件中，以便所有未来的 G28 命令使用该相位。
（因此，在未来的归位操作中，即使限位开关触发的时间略早或略晚，
Kalico 也将获得相同的位置。）

要为特定步进电机保存限位开关相位，请运行类似以下的内容：
```
ENDSTOP_PHASE_CALIBRATE STEPPER=stepper_z
```

对所有想要保存的步进器运行上面的命令。通常，
会在笛卡尔和 corexy 打印机上为 stepper_z 使用此功能，
在 delta 打印机上为 stepper_a、stepper_b 和 stepper_c 使用此功能。
最后，运行以下内容以使用数据更新配置文件：
```
SAVE_CONFIG
```

### 其他注意事项

* 此功能在 delta 打印机上最有用，以及在笛卡尔/corexy 打印机的 Z 限位开关上。
  可以在笛卡尔打印机的 XY 限位开关上使用此功能，但这并不特别有用，
  因为 X/Y 限位开关位置的轻微错误不太可能影响打印质量。
  在 corexy 打印机的 XY 限位开关上使用此功能无效
  （因为在 corexy 运动学上，XY 位置不是由单个步进器确定的）。
  在使用"probe:z_virtual_endstop"Z 限位开关的打印机上使用此功能无效
  （因为如果限位开关不在导轨上的静止位置，步进相位才不稳定）。

* 校准限位开关相位后，如果限位开关后来被移动或调整，
  则需要重新校准限位开关。从配置文件中移除校准数据并重新运行上述步骤。

* 为了使用此系统，限位开关必须足够精确以在两个"整步"内识别步进位置。
  因此，例如，如果步进器使用 16 微步，步距离为 0.005mm，
  则限位开关必须至少有 0.160mm 的精度。如果收到"Endstop stepper_z incorrect phase"
  类型的错误消息，则可能是由于限位开关精度不足。
  如果重新校准无法帮助，请通过从配置文件中移除限位开关相位调整来禁用它们。

* 如果使用传统步进控制的 Z 轴（如笛卡尔或 corexy 打印机上）
  以及传统床平整化螺钉，也可以使用此系统来安排每个打印层
  在"整步"边界上执行。要启用此功能，请确保 G-Code 切片机配置了
  是"整步"倍数的层高，在 endstop_phase 配置部分中手动启用 endstop_align_zero 选项
  （有关更多详细信息，请参阅 [配置参考](Config_Reference.md#endstop_phase)），
  然后重新平整床螺钉。

* 可以将此系统与传统（非 Trinamic）步进电机驱动器一起使用。
  但是，这需要确保每次微控制器重置时步进电机驱动器都会重置。
  （如果两者始终一起重置，则 Kalico 可以通过跟踪已命令步进器移动的总步数来确定步进相位。）
  目前，唯一可靠的方法是如果微控制器和步进电机驱动器仅由 USB 供电，
  并且 USB 电源来自在 Raspberry Pi 上运行的主机。在这种情况下，
  可以使用"restart_method: rpi_usb"指定一个 mcu 配置——该选项将安排
  微控制器始终通过 USB 电源重置进行重置，这将安排微控制器和步进电机驱动器
  一起重置。如果使用此机制，则需要手动配置"trigger_phase"配置部分
  （有关详细信息，请参阅 [配置参考](Config_Reference.md#endstop_phase)）。
