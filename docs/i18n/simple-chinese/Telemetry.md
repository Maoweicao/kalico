# Kalico 遥测

Kalico 包含一个可选的遥测系统，收集有关您打印机设置的匿名信息。本文档解释了什么是遥测、收集哪些数据、如何使用以及如何控制它。

## 什么是遥测？

遥测是收集匿名使用数据的过程，帮助 Kalico 项目了解用户如何使用该软件。这些信息对于以下方面非常有价值：

- 了解哪些功能最常用
- 识别哪些打印机配置最受欢迎
- 优先安排开发工作
- 改进与不同硬件设置的兼容性
- 为未来发展做出明智的决策

## 仅选择性加入

Kalico 的遥测系统是**严格选择性加入**的。当您首次运行 Kalico 时，系统会提示您启用或禁用遥测。在您明确启用之前，不会收集任何数据。

## 收集哪些数据

Kalico 遥测收集以下匿名信息：

### 机器标识
- 唯一的机器 ID（来自 `/etc/machine-id` 或存储在 `.machine-id` 中的生成 UUID）
- 此 ID 用于避免重复计算同一台机器，但无法追溯到您

### 系统信息
- 硬件架构（例如 `x86_64`、`arm64`）
- 来自 `/etc/os-release` 的操作系统信息（发行版名称、版本）
- Python 版本
- CPU 信息（仅型号名称）

### Kalico 软件信息
- Kalico 版本号
- Git 分支名称
- Git 远程 URL

### 配置信息
- **仅配置项名称**来自您的配置
- 不收集实际的配置值
- 示例：系统知道您有一个 `[printer]` 配置段，包含 `max_velocity` 等选项，但不知道您设置了什么值

### 启用的对象
- 所有已启用 Kalico 模块和插件的列表
- 这有助于了解正在使用哪些功能

## 不收集什么

Kalico 遥测**不**收集：

- 个人信息（姓名、电子邮件、IP 地址）
- 实际的配置值或参数
- 打印机日志或性能数据
- G-code 文件或打印历史
- 任何可以识别您个人身份的信息

## 如何控制遥测

### 查看您的遥测数据

您可以使用 `TELEMETRY_EXAMPLE` G-code 命令查看将发送的确切数据。这将在您的配置文件夹中保存一个名为 `telemetry.json` 的文件，包含将提交的确切数据。

### 启用遥测

如果您想通过分享匿名使用数据来帮助改进 Kalico：

1. 运行 `ENABLE_TELEMETRY` G-code 命令
2. 运行 `SAVE_CONFIG` 保存设置并重启打印机

### 禁用遥测

如果您不希望分享遥测数据：

1. 运行 `DISABLE_TELEMETRY` G-code 命令
2. 运行 `SAVE_CONFIG` 保存设置并重启打印机

## 数据处理

- 所有遥测数据发送到 `https://telemetry.kalico.gg/collect`
- 数据通过 HTTPS 安全传输
