# 多微控制器归位和探针

Kalico 支持一种机制，用于在一个微控制器（MCU）连接的限位开关上进行归位，同时其步进电机连接到不同的微控制器。此支持称为"多 MCU 归位"。当 Z 探针连接到不同的微控制器而不是 Z 步进电机时，也使用此功能。

此功能可能有用于简化接线，因为可能更方便地将限位开关或探针连接到更近的微控制器。但是，使用此功能可能会导致归位和探针操作期间步进电机的"超调"。

超调发生是由于监控限位开关的微控制器和移动步进电机的微控制器之间可能存在的消息传输延迟。Kalico 代码设计为将这种延迟限制在不超过 25ms。（激活多 MCU 归位时，微控制器发送定期状态消息并检查相应的状态消息是否在 25ms 内收到。）

例如，如果以 10mm/s 的速度进行归位，则可能发生高达 0.250mm 的超调（10mm/s * .025s == 0.250mm）。配置多 MCU 归位时应小心解释此类超调。使用较慢的归位或探针速度可以减少超调。

步进电机超调不应对归位和探针程序的精度产生不利影响。Kalico 代码将检测超调并在其计算中解释。但是，重要的是硬件设计能够处理超调而不对机器造成损害。

为了使用此"多 MCU 归位"功能，硬件必须在主机计算机和所有微控制器之间具有可预测的低延迟。通常，往返时间必须始终少于 10ms。高延迟（即使仅在短时间内）可能会导致归位失败。

如果高延迟导致失败（或检测到其他通信问题），Kalico 将引发"Communication timeout during homing"错误。

注意在同一微控制器上配置具有多个步进电机的轴（例如 `stepper_z` 和 `stepper_z1`）以使用多 MCU 归位。例如，如果限位开关在与 `stepper_z` 不同的微控制器上，则 `stepper_z1` 必须与 `stepper_z` 位于同一微控制器上。
