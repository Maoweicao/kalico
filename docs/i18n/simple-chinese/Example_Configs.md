# 示例配置

本文档包含为 Kalico GitHub 仓库贡献示例 Kalico 配置的指南（位于 [config 目录](../config/)）。

请注意 [Klipper 社区论坛](https://community.klipper3d.org) 也是查找和分享配置文件的有用资源。

## 指南

1. 选择合适的配置文件名前缀：
   1. `printer` 前缀用于由主流制造商销售的现成打印机。
   2. `generic` 前缀用于可能在许多不同类型打印机中使用的 3D 打印机主板。
   3. `kit` 前缀用于根据广泛使用的规范组装的 3D 打印机。这些"套件"打印机通常与普通"打印机"不同，因为它们不是由制造商销售的。
   4. `sample` 前缀用于配置"代码片段"，可以复制粘贴到主配置文件中。
   5. `example` 前缀用于描述打印机运动学。这种类型的配置通常只在添加新型打印机运动学代码时一起添加。
2. 所有配置文件必须以 `.cfg` 后缀结尾。`printer` 配置文件必须以年份后跟 `.cfg` 结尾（例如 `-2019.cfg`）。在这种情况下，年份是给定打印机销售的大约年份。
3. 不要在配置文件名中使用空格或特殊字符。文件名应仅包含字符 `A-Z`、`a-z`、`0-9`、`-` 和 `.`。
4. Kalico 必须能够启动 `printer`、`generic` 和 `kit` 示例配置文件而不出错。这些配置文件应添加到 [test/klippy/printers.test](../test/klippy/printers.test) 回归测试用例。在该测试用例中的适当部分内以字母顺序添加新配置文件。
5. 示例配置应为打印机的"原厂"配置。（要在主 Kalico 仓库中跟踪太多"自定义"配置。）同样，我们只为具有主流受欢迎程度的打印机、套件和主板添加示例配置文件（例如，至少应有 100 个处于活跃使用中）。考虑为其他配置使用 [Klipper 社区论坛](https://community.klipper3d.org)。
6. 仅指定给定打印机或主板上存在的设备。不要指定特定于您的特定设置的参数。
   1. 对于 `generic` 配置文件，应仅描述主板上的设备。例如，将显示配置部分添加到"通用"配置没有意义，因为没有办法知道主板是否会连接到该类型的显示器。如果主板有特定的硬件端口来方便可选外围设备（例如 bltouch 端口），则可以为给定设备添加"注释掉的"配置部分。
   2. 不要在示例配置中指定 `pressure_advance`，因为该值特定于灯丝，而不是打印机硬件。同样，不要指定 `max_extrude_only_velocity` 或 `max_extrude_only_accel` 设置。
   3. 不要指定包含主机路径或主机硬件的配置部分。例如，不要指定 `[virtual_sdcard]` 或 `[temperature_host]` 配置部分。
   4. 仅定义利用给定打印机特定功能或定义切片器常见发出的 G 代码的宏。
7. 如果可能，最好与现有配置文件使用相同的措辞、表述、缩进和部分排序。
   1. 每个配置文件的顶部应列出用户在"make menuconfig"期间应选择的微控制器类型。它还应该有对"docs/Config_Reference.md"的参考。
   2. 不要将字段文档复制到示例配置文件中。（这样做会造成维护负担，因为对文档的更新随后需要在许多地方进行更改。）
   3. 示例配置文件不应包含"SAVE_CONFIG"部分。如果需要，请将 SAVE_CONFIG 部分中的相关字段复制到主配置区域中的相应部分。
   4. 使用 `field: value` 语法而不是 `field=value`。
   5. 添加挤出机 `rotation_distance` 时，如果挤出机有齿轮传动机制，最好指定 `gear_ratio`。我们希望示例配置中的 rotation_distance 与挤出机中齿轮的周长相关联 - 通常在 20 到 35mm 的范围内。指定 `gear_ratio` 时，最好指定机制上的实际齿轮（例如，更倾向于 `gear_ratio: 80:20` 而不是 `gear_ratio: 4:1`）。有关更多信息，请参阅 [rotation distance 文档](Rotation_Distance.md#using-a-gear_ratio)。
   6. 避免定义设置为默认值的字段值。例如，不应指定 `min_extrude_temp: 170`，因为这已经是默认值。
   7. 如果可能，行应不超过 80 列。
   8. 避免向配置文件添加署名或修订消息。（例如，避免添加"此文件由 ..."创建的行。）将署名和更改历史记录放在 git 提交消息中。
8. 不要在示例配置文件中使用任何已弃用的功能。
9. 不要在示例配置文件中禁用默认安全系统。例如，配置不应指定自定义 `max_extrude_cross_section`。不要启用调试功能。例如，不应该有 `force_move` 配置部分。
10. Kalico 支持的所有已知主板都可以使用默认串行波特率 250000。不要在示例配置文件中推荐不同的波特率。

示例配置文件通过创建 GitHub"拉取请求"提交。请同时遵循 [贡献文档](CONTRIBUTING.md) 中的指示。
